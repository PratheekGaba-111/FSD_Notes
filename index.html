A Web Server is software and hardware that stores, processes and delivers web pages to clients. It handles HTTP requests and returns responses.

Static vs Dynamic Web Server

  • Static: Sends files exactly as stored. Only an HTTP server needed.
  • Dynamic: Has application server + database. Generates content on the fly (e.g., user-specific pages, login systems).

Important Question
Explain JavaScript Memory Management — life cycle, allocation, and garbage collection.
JavaScript automatically allocates memory when objects are created and frees it when no longer needed, through a process called Garbage Collection.

Memory Life Cycle

  1. Allocate — JS automatically allocates memory when values/objects are declared
  2. Use — Read from and write to the allocated memory
  3. Release — Garbage collector frees memory no longer referenced

Allocation Examples

const n = 123;         // allocates memory for a number
const s = "hello";    // allocates memory for a string
const o = { a: 1 };   // allocates for object + values
const a = [1,2,3];   // allocates for array

// Via function calls
const d = new Date();                    // allocates Date
const el = document.createElement("div"); // allocates DOM node

// Via methods that return new objects
const s2 = s.substr(0, 3);              // new string allocated
const a3 = a1.concat(a2);              // new array with merged items

The Garbage Collector uses the Mark-and-Sweep algorithm — it marks all objects reachable from the global root, then sweeps away everything else. Developers don't need to free memory manually in JS (unlike C/C++).

Explain React Forms — Controlled vs Uncontrolled Components.

Uncontrolled Component (DOM manages data via ref)

class App extends React.Component {
    constructor(props) {
        super(props);
        this.input = React.createRef();

        alert('Submitted successfully!');

                <label>Name: <input type="text" ref={this.input}/></label>
                <input type="submit" value="Submit"/>
            </form>
        );
    }
}

Controlled Component (React state manages data)

class App extends React.Component {
    constructor(props) {
        super(props);
        this.state = { value: '' };
    }
    handleChange(event) {
        this.setState({ value: event.target.value });
    }
    handleSubmit(event) {
        alert('Submitted: ' + this.state.value);
        event.preventDefault();
    }
    render() {
        return (
            <form onSubmit={this.handleSubmit.bind(this)}>
                <label>Name:
                    <input type="text"
                           value={this.state.value}
                           onChange={this.handleChange.bind(this)}/>
                </label>
                <input type="submit" value="Submit"/>
            </form>
        );
    }
}
Important Question
Explain React Router and Single Page Applications (SPA).
A Single Page Application (SPA) dynamically updates the current page rather than loading new pages from the server. React Router provides navigation between components without full page reloads.
// Install: npm i react-router-dom --save

import React, { Component } from "react";
import { Route, NavLink, HashRouter } from "react-router-dom";
import Home    from "./Home";
import Stuff   from "./Stuff";
import Contact from "./Contact";

class Main extends Component {
    render() {
        return (
            <HashRouter>
                <div>
                    <h1>Simple SPA</h1>
                    <ul>
                        <li><NavLink exact to="/">Home</NavLink></li>
                        <li><NavLink to="/stuff">Stuff</NavLink></li>
                        <li><NavLink to="/contact">Contact</NavLink></li>
                    </ul>
                    <div>
                        <Route exact path="/"       component={Home}   />
                        <Route       path="/stuff"   component={Stuff}  />
                        <Route       path="/contact" component={Contact}/>
                    </div>
                </div>
            </HashRouter>
        );
    }
}
export default Main;

The exact prop on the home route ensures it only matches / and not /stuff or /contact. NavLink automatically adds an active CSS class to the currently active link.

Important Question
Explain Spring MVC Architecture. What is DispatcherServlet? Describe the complete request flow.
Spring MVC is a web framework built on the MVC pattern. It separates concerns: Model (data/business logic), View (UI presentation), Controller (request handling). The central component is the DispatcherServlet.

MVC Components

  • Model — POJOs (Plain Old Java Objects) that hold application data
  • View — JSP/HTML files that render model data as output to the browser
  • Controller — Java class annotated with @Controller that handles HTTP requests, builds model, returns view name

Request Processing Flow — 6 Steps

  1. HTTP request arrives at DispatcherServlet (front controller)
  2. HandlerMapping is consulted — maps URL to the appropriate Controller
  3. Controller calls service/business logic, builds the Model, returns logical view name to DispatcherServlet
  4. DispatcherServlet asks ViewResolver to map the logical view name to actual view file
  5. View renders the model data as HTML
  6. Final HTML response sent back to the browser

web.xml — DispatcherServlet Configuration

<servlet>

public class Employee {
    private String id, name, address;

    public Employee(String id, String name, String address) {
        this.id = id; this.name = name; this.address = address;
    }
    // Getters and Setters for id, name, address
    public String getId()   { return id; }

EmployeeController.java (REST Controller)

@RestController   // = @Controller + @ResponseBody
public class EmployeeController {

    @Autowired
    private EmployeeService employeeService;

    // GET all employees
    @RequestMapping("/employees")
    public List<Employee> getEmployees() {
        return employeeService.getEmployees();
    }

    // GET one employee by id
    @RequestMapping("/employees/{empid}")
    public Employee getEmployee(@PathVariable("empid") String id) {
        return employeeService.getEmployee(id);
    }

    // POST — create new employee
    @RequestMapping(method = RequestMethod.POST, value = "/employees")
    public void addEmployee(@RequestBody Employee employee) {
        employeeService.addEmployee(employee);
    }

    // PUT — update employee
    @RequestMapping(method = RequestMethod.PUT, value = "/employees/{id}")
    public void updateEmployee(@RequestBody Employee e,
                                @PathVariable String id) {
        employeeService.updateEmployee(e, id);
    }

    // DELETE — remove employee
    @RequestMapping(method = RequestMethod.DELETE, value = "/employees/{id}")
    public void deleteEmployee(@PathVariable String id) {
        employeeService.deleteEmployee(id);
    }
}
AnnotationPurpose
@RestControllerMarks class as REST controller (combines @Controller + @ResponseBody)
@RequestMappingMaps URL + HTTP method to a handler method
@PathVariableExtracts value from URL segment (e.g., /employees/101 → id=101)
@RequestBodyDeserializes request JSON body into Java object
@AutowiredSpring auto-injects dependency

Explain Maven — Lifecycle, Phases, Goals, Plugins. How to build a Java application?
Maven is a Java project build and dependency management tool. It standardizes the build process using a pom.xml configuration file and divides the build into Lifecycles → Phases → Goals.

3 Maven Lifecycles

LifecyclePurpose
cleanRemove build artifacts from previous builds (deletes target/ folder)
defaultMain build lifecycle — compile, test, package, deploy
siteGenerate project documentation and reports

Default Lifecycle Phases (in order)

  1. validate — Verify project structure is correct
  2. compile — Compile all Java source files
  3. test — Run unit tests (using JUnit)
  4. package — Package compiled code into JAR or WAR
  5. deploy — Deploy packaged artifact to server/repository
Running a later phase automatically runs all prior phases. mvn package runs validate → compile → test → package.

Goals and Plugins

Goals are discrete tasks within a phase, provided by plugins. Format: pluginname:goalname

  • compiler:compile — compiles main source code
  • compiler:testCompile — compiles test source code

Common Maven Commands

mvn package        # Build JAR/WAR file
mvn clean          # Delete target/ folder

Databases & Deployment — SQL, Normalization, Spring JDBC, Agile, Cloud
Important Question
What is SQL? Explain DDL, DML, DQL, TCL with commands and examples.
SQL (Structured Query Language) is the standard language for relational databases. It is case-insensitive. Used with MySQL, Oracle, SQL Server, PostgreSQL etc.

DDL — Data Definition Language

Defines/alters database structure. Cannot be rolled back.

-- Create table
CREATE TABLE Student (
    ROLL_NO INT PRIMARY KEY,
    NAME    VARCHAR(20) NOT NULL,
    ADDRESS VARCHAR(50),
    PHONE   VARCHAR(15),
    AGE     INT
);

-- Add a column
ALTER TABLE Student ADD CGPA FLOAT;

-- Drop a table
DROP TABLE Student;

DML — Data Manipulation Language

Manipulates records. Can be rolled back.

INSERT INTO Student VALUES (1, 'RAM',    'DELHI',   '9455123451', 18);
INSERT INTO Student VALUES (2, 'RAMESH', 'GURGAON', '9652431543', 18);

UPDATE Student SET AGE = 20 WHERE ROLL_NO = 3;

DELETE FROM Student WHERE ROLL_NO = 4;

DQL — Data Query Language

-- Select all
SELECT * FROM Student;

-- Select specific columns with condition
SELECT ROLL_NO, NAME FROM Student WHERE ROLL_NO > 2;

-- Order by
SELECT * FROM Student ORDER BY AGE DESC;

-- Distinct values
SELECT DISTINCT ADDRESS FROM Student;

-- Aggregation
SELECT COUNT(ROLL_NO) FROM Student;  -- 4
SELECT SUM(AGE)       FROM Student;  -- 74
SELECT AVG(AGE)       FROM Student;  -- 18.5
SELECT MAX(AGE)       FROM Student;  -- 20
SELECT MIN(AGE)       FROM Student;  -- 18

-- GROUP BY
SELECT ADDRESS, SUM(AGE) FROM Student GROUP BY (ADDRESS);

TCL — Transaction Control Language

BEGIN TRANSACTION
    UPDATE employees SET empname = 'Bob' WHERE empid = '001';
    UPDATE employees SET empname = 'Bob' WHERE city = 'Hyderabad';
IF @@ROWCOUNT = 5
    COMMIT TRANSACTION     -- save changes permanently
ELSE
    ROLLBACK TRANSACTION   -- undo all changes
CommandPurpose
COMMITPermanently saves all DML changes in the transaction
ROLLBACKUndoes changes back to the last committed state
SAVEPOINT nameSets a checkpoint to rollback to
Important Question
What is Normalization? Explain its need, anomalies, and normal forms (1NF, 2NF, 3NF, BCNF).
Normalization is the process of organizing a relational database to reduce data redundancy and improve data integrity by splitting large tables into smaller, related tables.

Why Normalization? — The 3 Anomalies

  • Insert Anomaly: Cannot insert new data without adding unrelated data
  • Update Anomaly: Updating one fact requires updating multiple redundant copies
  • Delete Anomaly: Deleting a record accidentally removes other useful data

Normal Forms

NFCondition Required
1NFAll column values are atomic (indivisible). No repeating groups or arrays in cells.
2NFIs in 1NF + No partial dependency (every non-key attribute depends on the FULL primary key, not just part of it)
3NFIs in 2NF + No transitive dependency (non-key attributes depend only on the primary key)
BCNFStricter version of 3NF — every determinant is a candidate key

Cloud Computing is the on-demand delivery of IT resources (compute, storage, networking, software) over the internet with pay-per-use billing. Service models: IaaS, PaaS, SaaS.

4 Deployment Models

ModelDescriptionBenefitsLimitations
PublicOpen to everyone. Managed by AWS/Azure/GCP.No upfront cost, no hardware setup, easy scalingSecurity concerns, reliability risks
PrivateDedicated to one org. Hosted in-house or externally.High security, data privacy, supports legacyHigh cost, high maintenance
CommunityShared by orgs with common goals (e.g., hospitals).Cost-effective, meets industry standardsLimited bandwidth/storage, less popular
HybridCombination of public + private. Sensitive data in private, rest in public.Cost-effective, flexible, good securityComplex setup, specific use cases

Real-World Cloud Applications

  • Online Storage — Google Drive, Dropbox: store files anywhere
  • Backup & Recovery — Automatic backup, easy data recovery
  • Big Data Analytics — Store and analyze massive datasets
  • Testing & Dev — Scalable on-demand dev/test environments
  • E-Commerce — Manage products, customers, scale on demand
  • Education — e-Learning platforms, student portals
  • Healthcare — Patient records, remote doctor access
  • Entertainment — Netflix, Spotify: On-Demand Entertainment (ODE)

    }
}

import React, { Component } from 'react';

class RegistrationForm extends React.Component {
    constructor(props) {
        super(props);
        this.state = {
            name:        '',
            email:       '',
            isAgreed:    false,
            age:         18
        };
        this.handleInputChange = this.handleInputChange.bind(this);
        this.handleSubmit      = this.handleSubmit.bind(this);
    }

    handleInputChange(event) {
        const target = event.target;
        const value  = target.type === 'checkbox'
                         ? target.checked
                         : target.value;
        const name   = target.name;
        this.setState({ [name]: value });
    }

    handleSubmit(event) {
        alert('Registered: ' + this.state.name + ' | ' + this.state.email);
        event.preventDefault();
    }

    render() {
        return (
            <form onSubmit={this.handleSubmit}>
                <h2>Registration Form</h2>

                <label>Name:
                    <input type="text"  name="name"
                           value={this.state.name}
                           onChange={this.handleInputChange} />
                </label><br/>

                <label>Email:
                    <input type="email" name="email"
                           value={this.state.email}
                           onChange={this.handleInputChange} />
                </label><br/>

                <label>Age:
                    <input type="number" name="age"
                           value={this.state.age}
                           onChange={this.handleInputChange} />
                </label><br/>

                <label>
                    <input type="checkbox" name="isAgreed"
                           checked={this.state.isAgreed}
                           onChange={this.handleInputChange} />
                    I agree to the Terms and Conditions
                </label><br/>

                <button type="submit" disabled={!this.state.isAgreed}>
                    Register
                </button>
            </form>
        );
    }
}

export default RegistrationForm;

// EmployeeService.java — Business Logic
package com.example.rest;
import org.springframework.stereotype.Service;
import java.util.*;

@Service
public class EmployeeService {

    List<Employee> employeeList = new ArrayList<>(Arrays.asList(
        new Employee("e001", "Peter Parker",   "New York"),
        new Employee("e002", "Bruce Wayne",    "Gotham City"),
        new Employee("e003", "Clark Kent",     "Metropolis"),
        new Employee("e004", "Tony Stark",     "New York")
    ));

    public List<Employee> getEmployees() { return employeeList; }

    public Employee getEmployee(String id) {
        return employeeList.stream()
                .filter(e -> e.getId().equals(id))
                .findFirst().orElse(null);
    }

    public void addEmployee(Employee emp) { employeeList.add(emp); }

    public void updateEmployee(Employee emp, String id) {
        for (int i = 0; i < employeeList.size(); i++) {
            if (employeeList.get(i).getId().equals(id)) {
                employeeList.set(i, emp); break;
            }
        }
    }

    public void deleteEmployee(String id) {
        employeeList.removeIf(e -> e.getId().equals(id));
    }
}

package com.mrcet;
import java.util.List;
import javax.sql.DataSource;
import org.springframework.jdbc.core.JdbcTemplate;

public class StudentJDBCTemplate implements StudentDAO {

    private DataSource   dataSource;
    private JdbcTemplate jdbcTemplateObject;

    public void setDataSource(DataSource ds) {
        this.dataSource = ds;
        this.jdbcTemplateObject = new JdbcTemplate(ds);
    }

    // ── INSERT ──────────────────────────────────────────
    public void create(String name, Integer age) {
        String SQL = "INSERT INTO Student (name, age) VALUES (?, ?)";
        jdbcTemplateObject.update(SQL, name, age);
        System.out.println("Inserted: " + name);
    }

    // ── SELECT ALL ──────────────────────────────────────
    public List<Student> listStudents() {
        String SQL = "SELECT * FROM Student";
        return jdbcTemplateObject.query(SQL, new StudentMapper());
    }

    // ── SELECT ONE ──────────────────────────────────────
    public Student getStudent(Integer id) {
        String SQL = "SELECT * FROM Student WHERE id = ?";
        return jdbcTemplateObject.queryForObject(
            SQL, new Object[]{id}, new StudentMapper()
        );
    }

    // ── UPDATE ──────────────────────────────────────────
    public void update(Integer id, Integer age) {
        String SQL = "UPDATE Student SET age = ? WHERE id = ?";
        jdbcTemplateObject.update(SQL, age, id);
        System.out.println("Updated ID = " + id);
    }

    // ── DELETE ──────────────────────────────────────────
    public void delete(Integer id) {
        String SQL = "DELETE FROM Student WHERE id = ?";
        jdbcTemplateObject.update(SQL, id);
        System.out.println("Deleted ID = " + id);
    }
}
