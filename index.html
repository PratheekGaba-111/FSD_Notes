What is HTML? Explain its basic structure, tags, and key elements with examples.

Definition

HTML (HyperText Markup Language) is a standard markup language used to create web pages. It describes the structure of a web page using markup tags.
  • HyperText — Text that contains links to other documents/pages.
  • Markup Language — Uses tags to annotate/format text content.
  • Web Page — A document written in HTML, rendered by a browser. Can be static or dynamic.

What is a Web Server? Explain Client-Server architecture, HTTP methods and status codes.
A web server is a computer that stores web server software and a website's files (HTML, CSS, JS, images). It connects to the Internet and responds to HTTP requests from browsers.

How a Web Server Works

  1. Browser sends an HTTP request with a URL to the web server.
  2. The HTTP server software finds the requested file.
  3. It sends the file back to the browser via HTTP.
  4. If the file is not found, it returns a 404 Not Found response.

Static vs Dynamic Web Server

  • Static Web Server: Serves files exactly as stored. No processing. Simple.
  • Dynamic Web Server: Has an application server + database. Generates HTML content dynamically (e.g., user-specific pages).

Client-Server Architecture

A model where the client (browser) sends a request and the server provides the response. Communication uses the HTTP protocol.

HTTP Methods

MethodPurpose
GETFetch/retrieve data from the server
POSTSend new data to the server (create)
PUTUpdate existing data on the server
DELETERemove data from the server

HTTP Status Codes

CodeMeaning
200OK — Request succeeded
301Moved Permanently — Resource has a new URL
404Not Found — Resource doesn't exist
500Internal Server Error — Server-side problem

Q2 — 20 Marks
Explain JavaScript Memory Management — Memory Life Cycle, Allocation, and Garbage Collection.
JavaScript automatically allocates memory when objects are created and frees it when no longer used — this is called Garbage Collection.

Memory Life Cycle

  1. Allocate — Memory is allocated when values are declared
  2. Use — Read from / write to allocated memory
  3. Release — Free memory when no longer needed

Memory Allocation Examples

// Value initialization
const n = 123;          // number — allocates memory
const s = "hello";      // string — allocates memory
const o = { a: 1 };     // object — allocates memory for object + values
const a = [1, 2, 3];    // array — allocates memory

// Allocation via function calls
const d = new Date();   // allocates a Date object
const el = document.createElement("div"); // allocates DOM element

// Methods that allocate new values
const s2 = s.substr(0, 3);  // new string allocated

Garbage Collection

The most common algorithm is Mark-and-Sweep. The garbage collector periodically marks all objects still reachable from the root (global object), and sweeps away the unmarked objects. Developers in JavaScript don't need to manually free memory — the engine handles it automatically. This is different from low-level languages like C where you must call free() manually.

React JS — Components, State, Props, Router, Redux
Q1 — 20 Marks
Introduce React JS. Explain its features — JSX, Virtual DOM, Components, one-way data binding.
ReactJS is a declarative, efficient, flexible JavaScript library for building reusable UI components. Created by Jordan Walke at Facebook (released 2013). It handles the View layer (V in MVC). Applications are built from components.

Key Features of React

1. JSX (JavaScript XML)

JSX is a syntax extension that lets you write HTML-like code inside JavaScript. Preprocessors like Babel convert JSX to React.createElement() calls.

// JSX
const element = <div>Hello JavaTpoint</div>;

// Converted to:
React.createElement("div", null, "Hello JavaTpoint");

2. Components

An app is divided into small, independent, reusable pieces called components. Each component outputs a piece of HTML and has its own logic.

3. Virtual DOM

React maintains a virtual copy of the DOM in memory. When state changes, it re-renders the virtual DOM, computes the diff with the previous virtual DOM, and updates only the changed parts in the real DOM. This makes it very fast.

4. One-Way Data Binding

Data flows in one direction — from parent to child via props. This gives better control and makes the app easier to debug. Flux/Redux patterns support this.

5. Performance

Because of the Virtual DOM mechanism, React avoids unnecessary re-renders and is much faster than direct DOM manipulation.

Setup — Create React App

npx create-react-app myapp
cd myapp
npm start     # starts at http://localhost:3000
npm run build # production build

Explain React Components — Functional and Class Components, State, and Props with examples.

2. Class Components (Stateful)

Extend React.Component, must have a render() method. Can hold and manage local state.

class App extends React.Component {
    render() {
        return (
            <div>
                <h1>Hello React!</h1>
            </div>
        );
    }

State

State is an updatable structure that holds data about the component. Changes in state trigger a re-render. Use setState() to update.

class App extends React.Component {
    constructor() {
        super();
        this.state = { displayBio: false };
        this.toggleBio = this.toggleBio.bind(this);
    }
    toggleBio() {
        this.setState({ displayBio: !this.state.displayBio });
    }
    render() {
        return (
            <div>
                { this.state.displayBio ? <p>Bio content...</p> : null }
                <button onClick={this.toggleBio}>Toggle</button>
            </div>
        );
    }
}

Props

Props (Properties) are read-only data passed from parent to child component. Similar to function arguments. Immutable inside the component.

// Parent passes prop:
<App name="JavaTpoint" />

// Child uses prop:
class App extends React.Component {
    render() {
        return <h1>Welcome to {this.props.name}</h1>;
    }
}

Q3 — 20 Marks
Explain React Forms — Controlled and Uncontrolled Components. Explain React Router and SPA.

Uncontrolled Component

The DOM handles form data. Access values using ref. Simpler but less control.

class App extends React.Component {
    constructor(props) {
        super(props);
        this.input = React.createRef();
        this.handleSubmit = this.handleSubmit.bind(this);
    }
    handleSubmit(event) {
        alert("Submitted: " + this.input.current.value);
        event.preventDefault();
    }
    render() {
        return (
            <form onSubmit={this.handleSubmit}>
                <input type="text" ref={this.input} />
                <input type="submit" value="Submit" />
            </form>
        );
    }
}

Controlled Component

React state is the single source of truth. Every change goes through setState(). Better control over form data.

class App extends React.Component {
    constructor(props) {
        super(props);
        this.state = { value: '' };
    }
    handleChange(event) {
        this.setState({ value: event.target.value });
    }
    handleSubmit(event) {
        alert("Submitted: " + this.state.value);
        event.preventDefault();
    }
    render() {
        return (
            <form onSubmit={this.handleSubmit.bind(this)}>
                <input type="text"
                       value={this.state.value}
                       onChange={this.handleChange.bind(this)} />
                <input type="submit" value="Submit" />
            </form>
        );
    }
}

React Router & Single Page Applications (SPA)

A Single Page Application (SPA) loads a single HTML page and dynamically updates content as the user navigates, without full page reloads. React Router enables this by mapping URLs to components.
// Install: npm i react-router-dom --save

import { Route, NavLink, HashRouter } from "react-router-dom";
import Home from "./Home";
import Stuff from "./Stuff";
import Contact from "./Contact";

class Main extends Component {
    render() {
        return (
            <HashRouter>
                <div>
                    <ul>
                        <li><NavLink exact to="/">Home</NavLink></li>
                        <li><NavLink to="/stuff">Stuff</NavLink></li>
                        <li><NavLink to="/contact">Contact</NavLink></li>
                    </ul>
                    <div>
                        <Route exact path="/" component={Home}/>
                        <Route path="/stuff" component={Stuff}/>
                        <Route path="/contact" component={Contact}/>
                    </div>
                </div>
            </HashRouter>
        );
    }
}

The exact prop ensures the home route only matches / and not /stuff or /contact.

Q1 — 20 Marks
Explain Spring MVC Architecture. What is DispatcherServlet? How does request processing work?
Spring Web MVC is a framework based on the Model-View-Controller pattern. It separates application concerns: input logic (Controller), business logic (Model), and UI logic (View).

MVC Components

  • Model — Encapsulates application data. Usually POJOs (Plain Old Java Objects).
  • View — Renders model data as HTML output sent to the browser (JSP, Thymeleaf).
  • Controller — Processes user requests, builds the model, returns view name.

DispatcherServlet — The Front Controller

The DispatcherServlet is the central dispatcher in Spring MVC. All HTTP requests go through it. It delegates responsibilities to other components.

Request Processing Flow (Step by Step)

  1. HTTP Request arrives at the DispatcherServlet.
  2. HandlerMapping is consulted — maps the request URL to the appropriate Controller.
  3. Controller processes the request, calls service/business logic, builds the Model object, and returns a logical view name to DispatcherServlet.
  4. ViewResolver maps the logical view name to the actual view file (e.g., hello → /WEB-INF/jsp/hello.jsp).
  5. View renders the model data as HTML.
  6. Response is sent back to the browser.

Configuration — web.xml

<servlet>
    <servlet-name>HelloWeb</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <load-on-startup>1</load-on-startup>
</servlet>
<servlet-mapping>
    <servlet-name>HelloWeb</servlet-name>
    <url-pattern>*.jsp</url-pattern>
</servlet-mapping>

Defining a Controller

@Controller
public class HelloController {

    @RequestMapping(value = "/hello", method = RequestMethod.GET)
    public String printHello(ModelMap model) {
        model.addAttribute("message", "Hello Spring MVC!");
        return "hello";   // logical view name
    }
}

REST Controller

@RestController  // combines @Controller + @ResponseBody
public class EmployeeController {

    @Autowired
    private EmployeeService employeeService;

    @RequestMapping("/employees")
    public List<Employee> getEmployees() {
        return employeeService.getEmployees();
    }

    @RequestMapping("/employees/{empid}")
    public Employee getEmployee(@PathVariable("empid") String id) {
        return employeeService.getEmployee(id);
    }

    @RequestMapping(method = RequestMethod.POST, value = "/employees")
    public void addEmployee(@RequestBody Employee employee) {
        employeeService.addEmployee(employee);
    }

    @RequestMapping(method = RequestMethod.PUT, value = "/employees/{id}")
    public void updateEmployee(@RequestBody Employee e, @PathVariable String id) {
        employeeService.updateEmployee(e, id);
    }

    @RequestMapping(method = RequestMethod.DELETE, value = "/employees/{id}")
    public void deleteEmployee(@PathVariable String id) {
        employeeService.deleteEmployee(id);
    }
}

Explain SQL — DDL, DML, TCL with examples. Describe aggregation functions and GROUP BY.
SQL (Structured Query Language) is used to create, maintain, and retrieve data in relational databases. It is case-insensitive. Used in MySQL, Oracle, SQL Server, PostgreSQL etc.

DDL — Data Definition Language

Defines the structure of the database. Commands: CREATE, ALTER, DROP, RENAME

CREATE TABLE Student (
    ROLL_NO INT PRIMARY KEY,
    NAME    VARCHAR(20) NOT NULL,
    ADDRESS VARCHAR(50),
    AGE     INT
);

ALTER TABLE Student ADD PHONE VARCHAR(15);

DML — Data Manipulation Language

Manipulates data in tables. Commands: INSERT, UPDATE, DELETE

INSERT INTO Student VALUES (1, 'RAM', 'DELHI', 18);
UPDATE Student SET AGE = 20 WHERE ROLL_NO = 3;
DELETE FROM Student WHERE ROLL_NO = 4;

DQL — Data Query Language

-- Select all
SELECT * FROM Student;

-- Select with condition
SELECT ROLL_NO, NAME FROM Student WHERE ROLL_NO > 2;

-- ORDER BY
SELECT * FROM Student ORDER BY AGE DESC;

-- DISTINCT
SELECT DISTINCT ADDRESS FROM Student;

-- GROUP BY with aggregation
SELECT ADDRESS, SUM(AGE) FROM Student GROUP BY (ADDRESS);

Q2 — 20 Marks
Explain Normalization in SQL — what it is, why it is needed, and normal forms.
Normalization is the process of organizing database tables to reduce data redundancy and improve data integrity. It splits large tables into smaller related tables.

Why Normalization?

Without normalization, databases suffer from three anomalies:

  • Insert Anomaly — Cannot insert data without adding other unrelated data
  • Update Anomaly — Updating one record requires updating multiple redundant copies
  • Delete Anomaly — Deleting a record causes loss of other useful information

Normal Forms

Normal FormCondition
1NFAll columns have atomic (indivisible) values. No repeating groups.
2NFIs in 1NF + every non-key attribute is fully dependent on the entire primary key (no partial dependency)
3NFIs in 2NF + no transitive dependency (non-key attributes depend only on primary key, not on other non-key attributes)
BCNFStricter version of 3NF — every determinant must be a candidate key

Benefits of Normalization

  • Eliminates redundant data storage
  • Ensures data consistency and integrity
  • Makes updates/inserts/deletes efficient
  • Simplifies database design

<bean id="studentJDBCTemplate" class="com.mrcet.StudentJDBCTemplate">
    <property name="dataSource" ref="dataSource"/>
</bean>

1. INSERT

public void create(String name, Integer age) {
    String SQL = "INSERT INTO Student (name, age) VALUES (?, ?)";
    jdbcTemplateObject.update(SQL, name, age);
    System.out.println("Created: " + name);
}

2. SELECT (List All)

// StudentMapper.java
public class StudentMapper implements RowMapper<Student> {
    public Student mapRow(ResultSet rs, int rowNum) throws SQLException {
        Student s = new Student();
        s.setId(rs.getInt("id"));
        s.setName(rs.getString("name"));
        s.setAge(rs.getInt("age"));
        return s;
    }
}

// In StudentJDBCTemplate.java
public List<Student> listStudents() {
    String SQL = "SELECT * FROM Student";
    return jdbcTemplateObject.query(SQL, new StudentMapper());
}

3. UPDATE

public void update(Integer id, Integer age) {
    String SQL = "UPDATE Student SET age = ? WHERE id = ?";
    jdbcTemplateObject.update(SQL, age, id);
    System.out.println("Updated Record with ID = " + id);
}

4. DELETE

public void delete(Integer id) {
    String SQL = "DELETE FROM Student WHERE id = ?";
    jdbcTemplateObject.update(SQL, id);
    System.out.println("Deleted Record with ID = " + id);
}

MainApp.java

ApplicationContext context =
    new ClassPathXmlApplicationContext("Beans.xml");
StudentJDBCTemplate t =
    (StudentJDBCTemplate) context.getBean("studentJDBCTemplate");

t.update(2, 20);               // UPDATE

Describe different types of Cloud Computing Deployment Models and Real-World Applications.
Cloud Computing is the on-demand availability of IT resources (servers, storage, databases, software) over the Internet. Resources are delivered as services and billed on a pay-per-use basis.

Service Models

ModelFull FormDescription
IaaSInfrastructure as a ServiceProvides virtualized computing resources (VMs, storage, networking)
PaaSPlatform as a ServiceProvides platform for deploying applications (runtime, middleware)
SaaSSoftware as a ServiceDelivers software applications over the internet (Gmail, Office 365)

Cloud Deployment Models

1. Public Cloud

Accessible to the general public. Managed by cloud providers (AWS, Azure, GCP). Pay-per-use. Best for fluctuating demands.

✅ Minimal upfront cost, no hardware setup   ❌ Data security concerns, reliability issues

2. Private Cloud

Dedicated to a single organization. Hosted internally or externally. Greater control over data and security.

✅ High security, supports legacy systems   ❌ Higher cost, high maintenance

3. Community Cloud

Shared among organizations with common objectives (e.g., government agencies, hospitals). Managed internally or by a vendor.

✅ Cost-effective, meets industry standards   ❌ Shared bandwidth, limited capacity

4. Hybrid Cloud

Combination of public and private cloud. Sensitive data on private, less sensitive data on public. Supports "cloud bursting" — overflow to public during peak load.

✅ Cost-effective, flexible, more secure   ❌ Complex to set up and manage
